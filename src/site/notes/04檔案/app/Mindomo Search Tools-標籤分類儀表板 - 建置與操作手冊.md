## 1. 系統簡介
本系統是一個基於網頁的輕量級儀表板，透過 Mindomo API 自動抓取您的心智圖檔案。系統不使用傳統的資料夾分類，而是讀取檔案描述中的 **#Hashtag (標籤)** 來進行動態分類與篩選。

### 主要功能

- **自動標籤分類**：自動解析檔案描述中的 `#標籤` (如 `#工作`, `#學習`) 並生成側邊欄選單。
- **權限控管**：
    - **訪客模式**：預設僅顯示標籤包含 `#share` 的公開檔案。
    - **管理員模式**：輸入密碼後可查看帳號內的所有檔案。
- **進階搜尋**：支援邏輯運算子 (AND, OR, NOT) 進行精確查找。
- **多格式下載**：支援下載為 Markdown (.md)、Freemind (.mm) 及 PDF (.pdf)。

## 2. 建置與部署 (給管理員)
由於本系統為單一 HTML 檔案架構，部署非常簡單。
### 2.1 檔案準備
確保您擁有以下兩個檔案：
1. `mindomo_viewer.html` (主程式)
2. `linkc-w0.png` (Logo 圖檔，若使用網址連結則不需要本地圖檔)
### 2.2 部署方式

您可以選擇以下任一種方式使用：
- **方式 A：本機直接執行 (最簡單)**
    1. 將 `mindomo_viewer.html` 下載至電腦。
    2. 直接雙擊檔案，使用 Chrome 或 Edge 瀏覽器開啟即可使用。
- **方式 B：放置於網站伺服器 (推薦)**
    1. 將 html 檔案上傳至您的網站伺服器 (如 IIS, Apache, Nginx, 或 WordPress 的 uploads 資料夾)。
    2. 將網址提供給使用者即可。

### 2.3 參數修改

若未來需要變更設定，請使用「記事本」或程式碼編輯器 (如 VS Code) 開啟 `mindomo_viewer.html`，搜尋並修改以下變數：

- **API Token**：搜尋 `const HARDCODED_TOKEN`，修改引號內的字串。
- **管理員密碼**：搜尋 `const ACCESS_PASSWORD`，預設為 `'000000'`。
- **Logo 圖片**：搜尋 `link rel="icon"` 或 `img src=`，替換圖片網址。

> ⚠️ 安全性提醒：
> 
> 由於 API Token 與密碼直接寫在前端程式碼中，請勿將此頁面發佈於完全公開且不可信任的網路上。建議僅在公司內部網路或信任的社群中使用。

## 3. Mindomo 檔案設定規範 (給使用者)
為了讓儀表板能正確分類，請在 **Mindomo 網站** 上編輯檔案時，依照以下規範設定：
### 3.1 設定標籤
1. 開啟 Mindomo 檔案。
2. 點擊 **「檔案屬性 (File Properties)」** 或 **「摘要/描述 (Description/Notes)」** 欄位。
3. 在描述文字中加入標籤，格式為 `#` 加上文字。

**範例：**

> 這是一份關於 2025 年的行銷計畫草案... **#工作 #專案 #2025**

### 3.2 設定分享權限
若希望檔案在「訪客模式」下可見，請務必在描述中加入 **`#share`** 標籤。
- **私人檔案**：描述中**不要**加入 `#share`。
- **公開檔案**：描述中必須包含 `#share`。

## 4. 系統操作說明
### 4.1 瀏覽模式
- **訪客模式 (預設)**：進入系統時，您只能看到帶有 `#share` 標籤的檔案。標題旁會顯示「訪客模式」標記。
- **切換分類**：點擊左側側邊欄的標籤 (如 `#學習`)，右側列表會篩選出對應的檔案。
- **最近修改**：點擊左側「最近開啟」，會依照修改時間排序顯示前 20 筆資料。

### 4.2 管理員登入
若您需要查看所有檔案（包含未標示 #share 的私人檔案）：
1. 點擊左側選單最下方的 **「管理員登入」** 按鈕。
2. 輸入密碼：**`000000`**。-可依自己需求設定
3. 登入成功後，鎖頭圖示會解鎖，您將能看到所有資料。

### 4.3 搜尋功能

上方搜尋框支援強大的邏輯搜尋：

- **一般搜尋**：直接輸入文字 (如：`專案`)。
- **同時包含 (&)**：搜尋同時包含兩個條件 (如：`#工作 & #急件`)。
- **包含其一 (|)**：搜尋包含任一條件 (如：`#財務 | #會計`)。
- **排除 (!)**：搜尋不包含特定條件 (如：`#專案 & !#已完成`)。

### 4.4 檔案下載

在每個檔案卡片的右下角，提供三種格式下載：

- **MD**：Markdown 文件，適合匯入筆記軟體 (如 Obsidian, Notion)。
- **MM**：Freemind 格式，適合匯入其他心智圖軟體。
- **PDF**：便攜式文件格式，適合列印或閱讀。

### 4.5 除錯模式 (Debug)
若發現資料載入異常，可點擊左下角的 **「除錯模式」** 按鈕。系統會在頁面下方顯示原始 API 回傳的 JSON 資料，供技術人員檢查欄位是否正確。

## 5. 常見問題 (FAQ)

Q1: 為什麼我看不到某些檔案？
A:
1. 請確認您是否已登入「管理員模式」。若在訪客模式，只能看到有 `#share` 標籤的檔案。
2. 請確認 Mindomo 上的檔案描述欄位是否已正確儲存。

Q2: 下載按鈕點了沒反應？
A: 下載需要與 Mindomo API 通訊，請稍候約 1-2 秒。若長時間無反應，請檢查您的網路連線，或確認該檔案在 Mindomo 上是否已被刪除。

Q3: 為什麼搜尋 #tag 找不到東西？

A: 請確認輸入的標籤名稱與 Mindomo 上的完全一致（包含繁簡體）。此外，搜尋框輸入 # 時，系統會自動啟動邏輯搜尋模式，請確保語法正確。
